Introduction

Get access token

Access token is unique set of characters which identifies you, and allow you to use API.

If you are registered user, you can go to "Services token" tab in SEMSTORM system while logged in, and simply generate new token for yourself.

Access token allow to benefit from all API functionallity in the name of user who generated it, so dont share it with any third party! It should be handled with caution like passwords.

If you feel unsafe with your token - eg. you suspect that token could be obtained by unauthorised persons, you can regenerate your token, and get new one in same page you get your first. Just remember that previous token will be expired in moment you generate new one, so you have to replace it in all your scipts and functions which was using old one.

API results structure

Results have uniform structure. After calling API (in any way) client will receive response, if call was made using SEMSTORM PHP SDK API it will be php object, if calling directly by curl with .json prefix it will be encoded JSON string, eventhou every response will have structure which allows it's easy itnerpretation and data retrieve. Response object/array may contain:

"params": variable (object/assoc array) containing request aprameters - it will be params set by client and send to server, or -if optional- added by server automatically.

"result": variable (object/assoc array) containing request result(s) if no error occured.

"error": variable (object/assoc array) containing request info about any errors which occured, it dont exist if request was succesful. It have "message" field which holds information about reason of failure.

"_credits": variable (string/int) appears for most of results, showing how much request costs, it can be zero.

"_credits_left": variable (string/int) appears for most of results, showing how many credits left for current account.

Error message

If error occurs response will be send with proper http code (all success responses have code 200), and carry error information in body. Encoded body will contain variable keyed as "error" in which will be placed (at least this one) "message" string. This string have error code, and information text. See below possible error codes:

(400) Wrong input. Invalid or missing input data. Mostly additional info about requirements is displayed alongside error message.

(403) Access denied. Action you want to perform is impossible due to protection level.

(404) No entry for given id. Id dont exist - or is unavailable for current user.

(404) Undefined action. Callback was called but cannot determine which action to take.

(404) Not found. mostly followed by "Cound not find the controller.". You are calling wrong endpoint or wrong method - eg. using GET request in "monitoring-campaign/get-access" endpoint.

(409) Other transaction in progress. Access to data temporary denied. Something else is in critical section for this part of data. Try again later.

(500) Program exception catched. System exception occured. Exception message is display alongside error message.

(503) Service unavailable. Connection was disturbed. It can be whole system/connection temporary failure, or solr problem (see below)

(520) Something went wrong. Cant determine what, but something brokes on the way.

Misc. info

Changes in API are listed here and all apply to version 3 of SEMSTORM API.

Tags are obsolete. Eventhou tags placeholders ("tags" array in keywords, campaigns) still exists, tags functionallity is disabled, and is impossible to use. Well, small lost.

Problem with curl: If you get message "Unsupported request content type..." means you are using invalid content type. Curl examples below use JSON input/output format and we encourage to use this solution, if you want to communicate by JSON as well add -H "Content-Type: application/json" in your curl request (see here).

Variables types: Services backend is implemented in PHP, which does not require explicit type definition (see here), that is why in examples below all variables described as "int" can also be passed as string, and also returned types can differ from described below depend on content-type, and client environment.
If your software requires explicit variable type definition remember to cast results into proper variables.

Documentation

Below documentation describes SEMSTORM API part used for Monitoring functionallity. It is divided into sections describing each part of Monitoring.

Every functionallity caontains examples in curl and PHP. In PHP case, it is assumed that user have properly installed SEMSTORM PHP SDK library which can be fount here. For curl -as any other than prepared sdk library- remember that SEMSTORM API guarantee compatibility only for JSON formatted data.

Monitoring Campaign

MonitoringCampaign class instance

MonitoringCampaign class is used to perform operations on SEMSTORM Campaigns, which includes creating, retrieving, and modification of campaigns.
PHP example code:
include_once __DIR__ . '/autoload.php';
use SemstormApi\Semstorm;
use SemstormApi\Monitoring\MonitoringCampaign;
Semstorm::init( __ACCESS_TOKEN__ );
$monitoringCampaign = new MonitoringCampaign();

Create campaign

Endpoint: /monitoring/monitoring-campaign [POST]

Input: array ["domain"] string domain

Description: Creates new campaign.
Can create only "seo" type campaigns.

Output: object ["id"] string campaign id

Error:
Request: domain field is empty.
Limit: creating campaign require at least one free campaign limit.

PHP example code:

examples/Monitoring/CampaignCreate.php

Curl input:

curl -X POST -d '{ "domain": "example.com" }' http://api.semstorm.com/api-v3/monitoring/monitoring-campaign.json?services_token=__ACCESS_TOKEN__

Curl output "result":

{"id": "_ID_"}

Get detailed data about campaigns

Endpoint: /monitoring/monitoring-campaign/get-data [POST]

Input:
array["id"] string campaign id.
array["domains"] mixed array of domains or string with one domain or nothing then domain from campaign will be taken.
array["params"] assocc array with additional params.
array["params"]["type"] string type of data to retrieve, valid values are "organic", "paid" and "maps".
array["params"]["datemin"] string start date in format Ymd eg. 20160901.
array["params"]["datemax"] string end date in format Ymd eg. 20170530.
array["params"]["gap"] string date gap to aggregate data by, possible values are "daily", "weekly", "monthly".
array["params"]["filters"] array with filters.

Description: Get details about campaign keywords. It is data from "Daily rank tracking" tab in Monitoring/Keywords page in system.
In filters array you (optionally) specify your filters. It should be exact filters which are set in url when browsing monitoring page. Try this in system to see proper syntax.
For PHP library users: filters in input can be set using SemstormFilters class, which makes it a lot simplier. For details about SemstormFilters class see examples/Explorer/FiltersExplain.php and examples/Explorer/FiltersUsage.php files.

Output:
object["result"] array with query results divided by groups.
object["result"][_GROUP_ID_] object with group data divided by domains.
object["result"][_GROUP_ID_][_DOMAIN_] object with domain data divided by keywords.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_] object with keyword data.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["keyword"] object with keyword info.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["keyword"]["title"] string keyword name.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["keyword"]["volume"] int keyword search volume.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["keyword"]["cpc"] int keyword CPC value.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["data"] object with keyword data split by date (date format depends on date gap).
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["data"][_DATE_] object with keyword data split by date.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["data"][_DATE_][_ENGINE_] object with keyword data split by date.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["data"][_DATE_][_ENGINE_][_DEVICE_] object with keyword data split by date.
object["result"][_GROUP_ID_][_DOMAIN_][_KEYWORD_ID_]["data"][_DATE_][_ENGINE_][_DEVICE_] object with keyword data split by date.
object["params"] object with used params.
object["params"]["cid"] string campaign id.
object["params"]["domains"] array with domains.
object["params"]["datemin"] string start date (Ymd format).
object["params"]["datemax"] string end date (Ymd format).
object["params"]["gap"] string date gap, one of "daily", "weekly" or "monthly".
object["params"]["filters"] array with used filters.

Error:
Throttling: Solr throttling error.

PHP example code:

examples/Monitoring/CampaignData.php

Curl:

curl -X POST -d '{"id": _ID_, "params": {"datemin": "20170901", "datemax": "20171127", "gap": "monthly"}, "domains": ["example.com", "example2.pl"] }' http://api.semstorm.com/api-v3/monitoring/monitoring-campaign/get-data.json?services_token=__ACCESS_TOKEN__

Output:

{ "result": { (see php example for example result) } }

Get campaigns access

Endpoint: /monitoring/monitoring-campaign/get-access [POST]

Input: none, you will receive information about all campaigns you have access to.

Description: Returns information about campaigns you have access to, with type of access and infrmation about owner (e-mail address).
Existing permissions are:
"owner" - this account can modify everything in campaign, add and remove roles. Its reserved for account which created campaign, and is persistent.
"manager" - this account can do everything as "owner" but this role can be removed.
"readonly" - account with that permission can only see campaign and its statistics, and cannot do any changes.

Output: object [__CAMPAIGN_ID__] object with "access" where is your access level, and "owner" where is owner e-mail
object [__CAMPAIGN_ID__]["access"] string your access level
object [__CAMPAIGN_ID__]["owner"] string owner e-mail

Error:
none.

PHP example code:

examples/Monitoring/CampaignGetAccess.php

Curl:

curl -X POST http://api.semstorm.com/api-v3/monitoring/monitoring-campaign/get-access.json?services_token=__ACCESS_TOKEN__

Output:

{ "result": { (see php example for example result) } }

Set campaigns access

Endpoint: /monitoring/monitoring-campaign/set-access [POST]

Input: array [__CAMPAIGN_ID__] array
array [__CAMPAIGN_ID__]["set"] array keyed by campaign id which access are to be altered
array [__CAMPAIGN_ID__]["set"][]['mail'] string user mail
array [__CAMPAIGN_ID__]["set"][]['permission'] string permission level to set
array [__CAMPAIGN_ID__]["remove"] array of accounts to be removed from certain campaign
array [__CAMPAIGN_ID__]["remove"][]["mail"] string user mail

Description: Set access to campaigns, you need to have proper access level to modify permissions in campaigns.
This api endpoint is limited to one request per 10 seconds.
By this api call you can alter maximum 3 accounts at once. That means "set" array shouldn't have more than 3 records. There is no limit in removing access.
Proper permissions to give and take are "readonly" and "manager". One more permission named "owner" is reserved for account which created campaign and cannot be modified.
If you will try to grant permission by entering e-mail on which there is no registered account in SEMSTORM system, then the invitation e-mail to craete account will be send.

Output:
object [__CAMPAIGN_ID__]["set"][__USER_MAIL__] bool true if change was successful
object [__CAMPAIGN_ID__]["remove"][__USER_MAIL__] bool true if removal was successful

Error:
Request: too fast requesting, if any campaign id invalid (user might not have permissions) or if access updating failed. Access updating will fail if: you try to modify permissions in campaign you have "readonly" access, you try to remove or change permission level of the owner of campaign, one or more e-mail address is invalid.
Throttling: this request is specially limited to one per 10 seconds.

PHP example code:

examples/Monitoring/CampaignSetAccess.php

Curl:

curl -X POST -d '{ "__CAMPAIGN_ID__": { "set": [ (...) ], "remove": [ (...) ] }}' http://api.semstorm.com/api-v3/monitoring/monitoring-campaign/set-access.json?services_token=__ACCESS_TOKEN__

Output:

{ "result": { "__CAMPAIGN_ID__": { "set": [ (...) ], "remove": [ (...) ] } } }

Solr errors

Error with code 503 in most cases will concern solr connection. If error message will contain "Solr error. Data may be incomplete(...)" that means solr was overloaded and refused to sustain connection.
This is temporary problem, you can retry your request after few secons - just remember that very large request will require more time to wait.
